# BATTLE PLAN §17 — ручной прогон единого UI кампании

> Как РУКАМИ пройти весь поток §17 (сетап → seed → ветки → рабочий стол →
> эволюция), внося отклики самостоятельно. «Правду лаборатории» на каждом шаге
> считает хелпер `tools/response_helper.py` — вводишь координаты точки, получаешь
> отклики, переносишь их в форму. Истина хелпера = истина battle-теста
> (`src/verification/battle_truth.py`, единый источник — то же, к чему сходится
> пайплайн в `tests/unit/test_iteration13_battle.py`).

## 0. Подготовка

1. Запустить приложение:
   ```
   .venv/Scripts/python.exe -m streamlit run src/apps/streamlit_app.py
   ```
   Открыть вкладку **🧬 Кампания**.
2. Держать под рукой терминал для хелпера. Пример одного вызова:
   ```
   .venv/Scripts/python.exe tools/response_helper.py --world econ \
       A=0.25 B=0.25 C=0.25 D=0.25 T=0.5 P=0.5
   ```
   Хелпер печатает `strength/gloss/dry_time/whiteStrength/rho` (+ `price_состав`,
   `price_изд`). Именно эти пять свойств (rho — последнее) вносятся в столбцы
   «свойство (lab)» формы, В ТОМ ЖЕ ПОРЯДКЕ, что и отклики проекта.

### Мир прогона (`econ`, STEP 6)

- **Компоненты смеси:** `A, B, C, D` (доли, Σ=1).
- **Процесс-параметры:** `T, P`, границы **[0, 1]** (код куба — хелпер и истина
  ждут T,P в [0,1]; в сетапе кампании задаём нижние `0, 0`, верхние `1, 1`).
- **Отклики (свойства):** `strength, gloss, dry_time, whiteStrength, rho`
  (порядок важен — так их печатает хелпер и так их ждёт форма).
- **Цены компонентов (₽/кг):** A=95, B=200, C=23, D=315. Цена изделия
  `price_изд = price_состав·ρ` — хелпер её тоже печатает (для ценовой ноги ветки).
- **Истина (что «меряет» лаборатория), из `battle_truth.TRUTH_ECON`:**
  - `strength` = 6A+10B+2C+2D +5·A:T +3·C:T −3·T² +9·A*B +5·A*C +16·B*C +12·A*B*C +4·B*D
  - `gloss`    = 3A+7B+3C+2D +6P −4·P² +7·A*B +14·B*C +6·A*C +15·A*B*C
  - `dry_time` = −4A+2B+4C+D −4T −5·C:T +2·T²   (меньше — быстрее сохнет)
  - `whiteStrength` = 5A+6B+3C+D −8·C*D −5·A*D −4·B*D  (меньше — белее)
  - `rho`      = 0.7A+1.0B+1.7C+0.6D  (плотность; множитель цены изделия)

  (Пайплайн ЭТИХ формул не знает — он строит GP по вносимым точкам. Хелпер их
  знает, чтобы сыграть за лабораторию.)

| Процесс-параметры | `T, P` |

измерьте стартовый дизайн ниже».

## Шаг 2 — Стартовый дизайн seed с РУЧНЫМ вводом откликов (§17.4)

1. `N seed-точек` = **12**, `seed дизайна` = **1**. Нажать
   **«📐 Предложить seed-дизайн»**. Появится таблица из 12 строк с координатами
   `A,B,C,D,T,P` (заблокированы) и пустыми столбцами `*(lab)`.
2. **Для КАЖДОЙ строки** взять её координаты и посчитать отклики хелпером.
   Быстрее — выгрузить предложенные точки и прогнать пачкой:
   - вариант «по одной»:
     ```
     .venv/Scripts/python.exe tools/response_helper.py --world econ \
         A=<...> B=<...> C=<...> D=<...> T=<...> P=<...>
     ```
   - вариант «пачкой из CSV» (колонки-заголовки = `A,B,C,D,T,P`; значения из
     таблицы формы):
     ```
     .venv/Scripts/python.exe tools/response_helper.py --world econ --csv seed.csv
     ```
3. Вписать полученные `strength/gloss/dry_time/whiteStrength/rho` в
   соответствующие столбцы `*(lab)` (округление до 4 знаков — как печатает
   хелпер). Порядок столбцов = порядок откликов проекта.
4. Нажать **«💾 Зафиксировать seed (commit_seed)»**.

> Быстрый чек механики: кнопка **«🧪 Заполнить тестовыми (демо-оракул)»** заполнит
> Y демо-генератором (ManualOracle) — НЕ battle-истиной. Для честного battle-
> прогона используйте хелпер; демо-кнопка нужна лишь чтобы прощёлкать поток.

**Проверка:** «Seed зафиксирован: +12 точек (origin=seed), общая база = 12,
суррогаты обучены». Ниже открылись «➕ Создать ветку вручную» и «🧬 Эволюция схемы».

## Шаг 3 — Ручное создание веток (§17.5, Ш4)

Создаём 4 ветки STEP 6. Для каждой: имя, бюджет, цели (добавляются по одной
кнопкой «➕ Добавить цель в ветку»), опц. ценовая нога.

Общие поля ценовой ноги (одинаковы для всех веток):
- галка **«Задать ценовую ногу»**, `ρ-отклик` = **rho**,
  `Цены компонентов A,B,C,D` = **`95, 200, 23, 315`**, `верх цены` = **300**.

### Ветка `premium`
- Бюджет 30. Цели:
  - `strength` — вид `max`, low `2`, high `12`, вес `1`
  - `gloss` — вид `max`, low `1`, high `13`, вес `1`
- Ценовая нога: да (rho + цены выше).

### Ветка `economy`
- Бюджет 30. Цели:
  - `strength` — `max`, low `2`, high `12`, вес `1`
  - `dry_time` — `min`, low `-3`, high `3`, вес `1`
- Ценовая нога: да.

### Ветка `fast`
- Бюджет 30. Цели:
  - `dry_time` — `min`, low `-3`, high `3`, вес `1.5`
  - `gloss` — `target`, low `1`, high `13`, target `8`, вес `1`
- Ценовая нога: да.

### Ветка `white`
- Бюджет 30. Цели:
  - `whiteStrength` — `min`, low `0`, high `15`, вес `2`
  - `strength` — `max`, low `2`, high `12`, вес `1`
- Ценовая нога: да.

**Проверка:** каждая ветка создаётся; для веток с ρ печатается «канал занулён=…»
(если ρ не в целях — канал ALIVE/False; для white/economy ρ не в целях → ρ =
PRICE_INPUT).

## Шаг 4 — Рабочий стол ветки: добор с РУЧНЫМ вводом Y (§17.2/§17.5, Ш5)

Выбрать ветку в селекторе «Ветка (линза контекста)». В экспандере **«🛠 Рабочий
стол ветки»**:

1. `N точек` = **5**, `explore` = **0.2**. Нажать **«📐 Предложить точки
   (read-only)»**. Появится таблица кандидатов (координаты заблокированы).
2. Посчитать отклики этих 5 точек хелпером (по одной или CSV), вписать в
   столбцы `*(lab)`.
3. Нажать **«💾 Долить измеренные (commit_measured)»**. Ожидание: «Долито 5
   точек (origin=branch:{id}); d_best … → … (монотонно не убывает)»; ниже —
   §4-стоп (ceil_reached / not_economical / stagnation / продолжать).
4. Повторить 3–6 раундов на ветку (или пока §4-стоп не скажет «потолок/невыгодно»).

**Что наблюдать (сходимость — суть battle-проверки):**
- `d_best` ветки **растёт** от раунда к раунду и не убывает (И-1).
- Рекомендованный рецепт (см. редактор целей / M8-argmax) идёт к «правильному»:
  - **`white`** держит **D > 0** (D — отбеливатель, без него whiteStrength не
    снизить: в истине `−8·C*D −5·A*D −4·B*D`);
  - **`economy`** тянет к дешёвому составу (C=23 дёшев и повышает ρ; цена
    изделия падает) при сохранении strength через B-синергии;
  - **`premium`** держит **B** (сильные B-синергии в strength/gloss:
    `16·B*C`, `12·A*B*C`);
  - **`fast`** держит **A** (в истине `dry_time` A=−4 — быстрая сушка) и
    открывает **T** (T=−4, C:T=−5 ускоряют сушку).
- Сверить с эталоном: прогнать `tests/unit/test_iteration13_battle.py::
  test_battle_step6_item_price_economy_white_branch -s` — печатает аналитические
  оптимумы и рецепты пайплайна (те же цели), это референс, к которому ручной
  прогон должен быть близок.

## Шаг 5 — Эволюция схемы в любой момент (§16.2/§17.6, Ш6)

Экспандер **«🧬 Эволюция схемы»** (проверка, что живой проект меняется без потери
истории, И-1):

- **Ввести новый отклик**: имя, напр. `viscosity`. Ожидание: «версия поднята; у
  старых точек значение MISSING». (Хелпер это свойство не считает — новый отклик
  меряется только на НОВЫХ точках; для battle-прогона можно не гнать дальше.)
- **Подвинуть границы области** (relax/restrict): ось `C`, нижняя `0`, верхняя
  `0.2`, намерение `restrict` → «область обновлена (история цела)». Затем relax
  обратно `0..1`. Проверка: база точек не урезается, версия при region-move НЕ
  растёт.

## Приложение — быстрые команды хелпера

```
# одна точка (econ, по умолчанию)
.venv/Scripts/python.exe tools/response_helper.py A=0.25 B=0.25 C=0.25 D=0.25 T=0.5 P=0.5

# 3-компонентный фазовый мир {A,B,C}
.venv/Scripts/python.exe tools/response_helper.py --world 3comp A=0.3 B=0.3 C=0.4 T=0.5 P=0.5

# пачка точек из CSV (заголовки = A,B,C,D,T,P)
.venv/Scripts/python.exe tools/response_helper.py --world econ --csv seed.csv
```

Замечания:
- процесс `T,P` — в коде **[0,1]** (как границы процесса в сетапе);
- пропущенные `T,P` хелпер берёт как `0.5` (центр куба);
- если Σ долей смеси ≠ 1 — хелпер предупредит (посчитает как введено).
