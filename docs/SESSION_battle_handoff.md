# test_iteration13_battle — handoff для архитектурной сессии

Краткая выжимка по боевому тесту `tests/unit/test_iteration13_battle.py`
(`test_battle_branches_converge_to_analytic_optimum`). Акцент — на **целевых
математических функциях** (истина + желательность), а не на программных
методах. Источник всех чисел — код модулей (проверено чтением, не по памяти).

---

## 1. Область

- **MIXTURE** {A, B, C}, доли, ограничение симплекса `A + B + C = 1`.
- **PROCESS** {T, P}, в коде `[0, 1]` каждая.
- Составные координаты `Xc = [A, B, C, T, P]`.
- baseline (для «закрытых» фазой переменных): `[1/3, 1/3, 1/3, 0.5, 0.5]`.

## 2. Истинные функции свойств η(x, z) — «синтетическая физика»

Общая форма истины (Scheffé по составу + RSM по процессу + кросс), реализуется
через `MixtureProcessTruth.true(Xc) = M(Xc) @ β`, где `M` — модельная матрица
термов из `design.block_model.build_model_terms` для схемы истины
(**cubic** mixture + **quadratic** process + full-cross):

```
η = Σ βi·xi + Σ βij·xi·xj            (Scheffé mixture, без intercept)
  + Σ γk·zk + Σ γkl·zk·zl            (RSM process, код [0,1])
  + Σ δik·xi·zk                       (кросс mixture×process)
```

Конкретные ненулевые коэффициенты по свойствам (всё остальное = 0):

| свойство   | функция истины η (A,B,C — доли; T,P — код [0,1]) |
|-----------|---------------------------------------------------|
| **strength** | `10·A + 6·B + 2·C + 2·(A·T) + 8·(A·B·C) − 3·T²` |
| **gloss**    | `4·A + 10·B + 1·C + 6·P − 4·P² + 6·(A·B·C)`      |
| **dry_time** | `3·A + 3·B + 3·C − 6·T + 6·(A·B·C)`  (линейная часть = 3, т.к. ΣA=1) |
| **price**    | `1·A + 2·B + 5·C`  (C — дорогой компонент)        |

Шум: `noise_sd = 0.0` (чистая истина — оптимум детерминирован).

### Намеренная мисспецификация (суть стресс-теста)

Модель пайплайна — **QUADRATIC** Scheffé (тренд/скрининг), а истина содержит
термы ВНЕ quadratic-тренда, которые обязан ловить только GP-остаток (kernel):

- `A·B·C` — специальная кубическая синергия состава (degree 3);
- `T²`, `P²` — чистые квадраты процесса (в Scheffé-quadratic их нет).

Парные термы (`A·B`, `A:T`, `T·P`, …) — degree 2, лежат ВНУТРИ тренда (не стресс).

## 3. Функции желательности (как заданы целевые значения)

Derringer–Suich, на свойство `d_i(y) ∈ [0,1]` (форм. экспонента `s=1` по
умолчанию → линейные рампы):

```
max    : d = 0,                       y ≤ low
             ((y−low)/(high−low))^s,   low < y < high
             1,                        y ≥ high
min    : d = 1,                       y ≤ low
             ((high−y)/(high−low))^s,  low < y < high
             0,                        y ≥ high
target : двусторонний пик в `target`; 0 вне [low, high],
             ((y−low)/(target−low))^s    при y ≤ target,
             ((high−y)/(high−target))^s2 при y > target
```

Свёртка ветки — **взвешенное геометрическое среднее** с жёстким вето:

```
d_overall = ( Π_i d_i^{w_i} )^{1/Σ w_i};   если любое d_i = 0 → d_overall = 0
```

### Цели трёх веток (внутри ветки цели «спорят»)

| ветка | целевые функции (свойство: kind[low,high]{target} weight) |
|-------|------------------------------------------------------------|
| **premium** | strength: max[2,12] w1; gloss: max[1,13] w1; price: min[1,5] w0.3 |
| **economy** | price: min[1,5] **w2**; strength: max[2,12] w1; dry_time: min[−3,3] w1 |
| **fast**    | dry_time: min[−3,3] w1.5; gloss: **target[1,13] target=8** w1; price: min[1,5] w0.5 |

Цена присутствует в каждой ветке. Конфликты: premium тянет A (strength) против
B (gloss); economy жёстко минимизирует цену против strength/сушки; fast держит
gloss у целевого 8 против минимизации цены.

## 4. Эталоны (verification, считаются по ИСТИНЕ, не по суррогату)

- `branch_optimum(truth, goal)` — **глобальный** оптимум ветки над всей областью
  (плотный скан 20000 точек + локальное уточнение SLSQP с Σx=1 и боксами).
- `branch_optimum_masked(truth, goal, baseline, mixture_free, process_free)` —
  **потолок фазы**: max desirability по истине при варьировании только
  открытых маской координат (скан 40000, без SLSQP).

## 5. Последовательность теста

1. Для каждой ветки считается глобальный `branch_optimum` — цель сходимости.
2. Создаётся `MixtureProcessRunner` (одна общая модель = словарь GP-суррогатов
   на свойство; ветки — контейнеры намерения, общая база точек с origin-тегами).
3. **Фаза 1** (свободны A,B; C,T,P на baseline): `seed_initial(n=18)` — общий
   старт для всех веток; затем один branch-раунд (n=5) на ветку → снимок `d1`.
4. **Фаза 2** (+T): один branch-раунд (n=5) на ветку → `d2`.
5. **Фаза 3** (+C,+P, полная свобода): один branch-раунд (n=7) на ветку → `d3`.
6. Считаются потолки фаз `branch_optimum_masked` → `ceil = [c1,c2,c3]`.
7. Проверки фаз: монотонность d1≤d2≤d3; d3≤d_opt; d3≥0.6·d_opt; монотонность
   потолков; пайплайн не превышает потолок своей фазы; раскрытие помогло ≥2
   веткам; origin: seed==18, branch≥10.
8. **Цена/качество (новое):** в фазе 3 дозабор продолжается (branch-раунды n=5),
   пока ветка не застрянет (`is_stagnating`, patience=3, min_delta=5e-3) ИЛИ не
   исчерпает бюджет (60). Считается стоимость достижения 80/90/95% потолка ф3.
9. **Итог после дозабора (новое):** финальные рецепты pipeline x* vs аналитика +
   свойства по истине (+Δ цена).
10. Проверки дозабора (робастные): `d_final ≥ d3`; `d_final ≤ потолок ф3 + 0.03`.

### Соответствие M-уровням REBUILD_SPEC
- **M6** — `GPExpert` (mean=quadratic Scheffé, kernel=Matérn5/2) на КАЖДОЕ
  свойство. Это GP-вариант, **без MoE/GMM-gating** (K=1).
- **M7** — branch acquisition `acq = (1−explore)·d_pred + explore·σ̄_n` →
  отбор разнесённых кандидатов → измерение оракулом → дозапись → refit M6.
- **M8** — **усечён**: используется только функция desirability для оценки
  ИЗМЕРЕННЫХ точек; `x_best` = лучшая измеренная точка. Полного M8
  (мультистартовый `argmax` desirability по суррогату на симплексе +
  стоимостный член, как в `optimize_desirability`) в раннере НЕТ.

### Выводы для архитектуры

- **Убывающая отдача:** 80–90% потолка достигается уже ~на 17 опытах ветки
  (первый раунд полной свободы). Дальнейший дозабор почти удваивает базу
  (69→134) ради +1–3 п.п. desirability.
- **desirability маскирует физику:** малый прирост d_overall (геом. среднее
  насыщается) скрывает крупный выигрыш по отдельным свойствам — premium
  Δцена +10%→−3%, dry 3.0→1.9; fast Δцена +39%→+4%.
- **economy реально застряла на 94%:** её аналитический оптимум — в вершине
  симплекса (A=1) при T=1; масочный СЛУЧАЙНЫЙ семплер M7 туда почти не
  попадает. Кандидаты на доработку: (а) acquisition с тягой к вершинам/границам
  симплекса; (б) ввод настоящего M8-argmax (`optimize_desirability` уже есть в
  `src/optimize/desirability.py` — мультистарт + локальное уточнение в
  псевдокомпонентах) в branch-раунд вместо «лучшей измеренной точки».
