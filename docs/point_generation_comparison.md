# Point Generation Comparison: test_parameter_recovery vs JMP MixtureDesign45Runs1Order

## Overview

This document compares how points are generated in our `test_parameter_recovery.py` script versus the actual JMP design from `MixtureDesigh45Runs1Order.xlsx`.

## Summary of Approaches

### JMP Design (Actual - from MixtureDesigh45Runs1Order.xlsx)
- **Method**: D-Optimal algorithm with strategic replication
- **Total Runs**: 45
- **Approach**: Uses optimization to maximize D-efficiency while ensuring good coverage
- **Characteristics**: Real-world design generated by JMP Pro software

### Our Design (test_parameter_recovery.py)
- **Method**: Pattern-based sparse design mimicking JMP strategy
- **Total Runs**: 45 
- **Approach**: Systematic generation using predefined sparsity patterns
- **Characteristics**: Attempts to reverse-engineer JMP's strategy based on observed patterns

### 2. Generation Algorithm Differences

#### JMP Approach (D-Optimal with Replication)

From analyzing `MixtureDesigh45Runs1Order.xlsx`:

```python
# JMP uses D-optimal algorithm:
# 1. Start with candidate set of all possible mixture points
# 2. Optimize point selection to maximize det(X'X)

# 4. Result: Mathematically optimal design for parameter estimation

# Example JMP points (from actual file):
```

**Key Features:**
- Uses optimization algorithm (coordinate exchange or similar)
- Includes **duplicate points** for statistical reasons (replication)
- Points may have very precise decimal values (result of optimization)
- Distribution adapts to specific model requirements

## Key Insights

### 1. **Replication Strategy** (BIGGEST DIFFERENCE!)

**JMP:**
- Strategically replicates certain points
- Example: Interior centroid appears 3 times
- Replication improves:
  - Statistical power for key regions
  - Error estimation
  - Model validation capability

**Our Design:**
- Every point is unique
- No strategic replication
- May miss statistical benefits of replication

### 2. **Optimization vs Patterns**

**JMP:**
```
Uses optimization → D-efficiency maximized
Iteration: Try points → Calculate D-eff → Keep best → Refine
Result: Mathematically optimal point placement
```

**Our Design:**
```
Uses patterns → Approximate good coverage
Strategy: Apply patterns → Shuffle → Combine
Result: Good empirical coverage, not mathematically optimal
```

### 3. **Sparsity Goals**

Both achieve ~80% sparsity (2+ zeros), but:
- **JMP**: Sparsity emerges from optimization
- **Our**: Sparsity is the primary design constraint

### 4. **Point Precision**

**JMP**: Optimized values like `0.332874003194798...`
**Our**: Pattern values like `0.333...` or `0.500...`

## Conclusions

### Similarities ✅
1. Both target ~80% sparsity
2. Both include all vertex points
3. Both use multiple edge ratios
4. Both perform well on main effects and 2-way interactions

### Differences ⚠️
1. **JMP uses D-optimal optimization; we use patterns**
2. **JMP includes strategic replication; we don't**
3. JMP has precise optimized values; we have pattern values
4. JMP adapts to model; we use fixed distribution

### Recommendation 💡

For production use:
1. **Keep our pattern-based approach** for speed and simplicity
2. **Add strategic replication** of key points (centroid, important edges)
3. **Consider D-optimal refinement** as optional post-processing step
4. **Make sparsity distribution configurable** rather than fixed

Example improved code:
```python
def generate_jmp_sparse_design_v2(n_runs, n_components=5, 
                                   allow_replication=True,
                                   refine_with_d_opt=False):
    """
    Improved version combining pattern-based + optimization:
    1. Generate pattern-based design (80% of points)
    2. Replicate key points (10% of points)
    3. Optionally refine with D-optimal (10% of points)
    """
    # Pattern generation (fast)
    base_design = pattern_based_generation(n_runs * 0.8)
    
    # Replication (improve power)
    if allow_replication:
        key_points = identify_key_points(base_design)
        replicated = replicate_points(key_points, n_runs * 0.1)
        base_design.extend(replicated)
    
    # D-optimal refinement (improve efficiency)
    if refine_with_d_opt:
        refined = d_optimal_refinement(base_design, n_runs * 0.1)
        base_design.extend(refined)
    
    return np.array(base_design[:n_runs])
```
